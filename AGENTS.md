## Commands

### Run full phase (5 parallel browsers)
```bash
cd P-1
python run_all.py --phase 1    # Priority 1
python run_all.py --phase 2    # Priority 2
python run_all.py --phase 3    # Priority 3
```

### Run single batch manually
```bash
python scraper.py --input p1_batch_1.txt --output output/p1_batch_1.csv --profile profiles/p1_batch_1
```

### Merge batch CSVs into final file
```bash
python merge.py
# Reads output/p*_batch_*.csv → dedup (name+phone+address) → priority_final.csv
```

### Merge specific priority
```bash
python merge.py --pattern "p1_batch_*.csv" --output p1_final.csv
```

### View progress in browser
```
# Open view.html in any browser
# Shows live scraping dashboard (auto-refreshes every 10s)
# Also works as CSV drag-drop viewer for any exported file
```

## Architecture

```
run_all.py ──┬── scraper.py (batch 1) ──→ output/p1_batch_1.csv
             ├── scraper.py (batch 2) ──→ output/p1_batch_2.csv
             ├── scraper.py (batch 3) ──→ output/p1_batch_3.csv
             ├── scraper.py (batch 4) ──→ output/p1_batch_4.csv
             └── scraper.py (batch 5) ──→ output/p1_batch_5.csv
                         ↓
                  merge.py ──→ priority_final.csv
```

Each scraper.py:
1. Reads queries from `--input`
2. Skips queries in `.done` file (resume support)
3. Launches Playwright with persistent Chrome profile (`launch_persistent_context`)
4. For each query: loads Google Maps search → scrolls feed (~60-180 results) → clicks each card → extracts details → saves CSV
5. Writes query to `.done` file after completion

## Dedup Strategy
- **Per-batch**: dedup on `(name, phone, address)` during incremental saves
- **Final merge**: dedup on `(name, phone, address)` across all batches

## Conventions
- **NEVER touch** `google_maps_leads.csv` (legacy export)
- Batch CSVs → `output/p{phase}_batch_{n}.csv`
- Profiles → `profiles/p{phase}_batch_{n}` (auto-created by Playwright)
- Done files → `output/p{phase}_batch_{n}.done`
- Log files → `output/p{phase}_batch_{n}.log`
- Progress state → `output/progress.json`
- Final merged → `P-1/priority_final.csv`

## P-2 (Docker-based)

Uses [gosom/google-maps-scraper](https://github.com/gosom/google-maps-scraper) instead of custom Playwright.

```
P-2/
├── p2_queries.txt      # All 324 P2 queries (combined from P-1 batches)
├── run.bat              # Windows CMD runner
├── run.ps1              # PowerShell runner
└── run.sh               # Git Bash / WSL runner
```

### Run P-2
```bash
cd P-2
./run.sh                   # Git Bash / Linux / WSL
# or: cmd //c run.bat      # CMD
# or: powershell -ExecutionPolicy Bypass -File run.ps1  # PowerShell
```

### Output
Results land in `gmaps-output/p2_results.csv`.

Column schema is different from P-1 (uses the gosom tool's schema — 36 fields including emails, coordinates, reviews, etc.).

### Merge into final
```bash
python merge.py --pattern "gmaps-output/*.csv" --output P-2/p2_final.csv
```
